## Trap Doors

- `purge-update-cache.js` — INVARIANT: purge root MUST equal validated `getExtensionRoot()` resolution (`EXTENSION_DIR` only when install-root sentinel exists, else fallback to `~/.claude/pickle-rick`); `update-check.json` and `deploy-audit.log` are written by `extension/src/bin/check-update.ts` and `install.sh` to that path. BREAKS: purge no-ops in production when raw env paths or wrong defaults miss the live cache root; stale auto-updater cache poisons the next update check. ENFORCE: extension/tests/purge-update-cache.test.js. PATTERN_SHAPE: `process\.env\.EXTENSION_DIR\s*\|\||path\.join\(os\.homedir\(\), '\.claude', 'pickle-rick'\)`.
- `release-gate.sh` — INVARIANT: release tag semver MUST equal `extension/package.json` version before publish and after download verification. BREAKS: updater caches a phantom latest version and re-downloads the same tarball forever. ENFORCE: extension/tests/release-gate.test.js. PATTERN_SHAPE: `--pre-tag|--post-tag` path without tag-name-vs-package-version parity check; INVARIANT: release gating MUST resolve `extension/package.json` and git tag contents from the active worktree root, not the caller's nested cwd. BREAKS: `bash ../bin/release-gate.sh` from `extension/` or another repo subdir false-fails before validating a real release tag. ENFORCE: extension/tests/release-gate.test.js. PATTERN_SHAPE: `PKG_PATH=\"extension/package\.json\"|git show \"\\$tag:\\$PKG_PATH\"` without `git rev-parse --show-toplevel` root resolution; INVARIANT: post-tag verification MUST select the unique installable tarball by archive shape and require both `extension/package.json` and `install.sh`. BREAKS: sidecar assets or partial payloads false-green the release while updater install later fails on missing runtime files. ENFORCE: extension/tests/release-gate.test.js. PATTERN_SHAPE: `tar -tzf .*extension/package\.json` without a same-archive `install\.sh` sentinel check; INVARIANT: post-tag verification MUST download release assets via `gh release download --pattern '*.tar.gz'`, never the source archive flag. BREAKS: the gate validates a GitHub source archive instead of the shipped installer payload and can false-green a broken release asset. ENFORCE: extension/tests/release-gate.test.js. PATTERN_SHAPE: `gh release download .* (-A|--archive) '?tar\.gz'?`.
- `verify-bundle.js` — INVARIANT: every `bundle/ac-dr-*.json` producer MUST emit the canonical bundle schema (`ac_id`, `checked_at`, `checker`, `checker_version`, `evidence`, `failure_reason`, `remediation_hint`). BREAKS: `verifyBundle()` rejects real session artifacts and the deploy-reversion bundle gate goes red before evaluating behavior. ENFORCE: extension/tests/verify-bundle.test.js, extension/tests/pipeline-runner.test.js. PATTERN_SHAPE: `write(StateFile|FileSync)\(.+bundle\/ac-dr-[^']+\.json`; INVARIANT: single-AC verification MUST normalize refined AC aliases to the canonical bundle artifact id before resolving `bundle/ac-dr-*.json`. BREAKS: `--ac AC-DR-15` false-fails against missing `ac-dr-15.json` while the full-bundle path expects `ac-dr-pre-flight.json`. ENFORCE: extension/tests/verify-bundle.test.js. PATTERN_SHAPE: `options\.ac\s*\?\s*\[options\.ac\]` without refined-id normalization; INVARIANT: single-AC verification MUST reject unknown requested AC ids before reading bundle files. BREAKS: typoed or stale `--ac` values false-pass against arbitrary `bundle/ac-dr-*.json` payloads outside the contract. ENFORCE: extension/tests/verify-bundle.test.js. PATTERN_SHAPE: `options\.ac` validation path without `EXPECTED_BUNDLE_AC_IDS` membership after canonicalization.
- `verify-recapture-fired.js` — INVARIANT: AC-DR-02 MUST match `baseline_recapture_attempted` only inside the latest anatomy window and MUST NOT pin a fixed iteration. BREAKS: stale runs or later-iteration recaptures false-pass or false-fail the bundle. ENFORCE: extension/tests/verify-recapture-fired.test.js. PATTERN_SHAPE: `iteration\s*===\s*1|findMatchingEvent\(activity,\s*anatomyWindows\(state\.history\)\)` without latest-window-only filtering; INVARIANT: session state reads for AC-DR-02 MUST use `StateManager.read(statePath)` so orphan tmp recovery promotes the live snapshot before window matching. BREAKS: recoverable `state.json.tmp.<pid>` sessions misreport `state-unreadable` and fail the bundle even when the recapture event exists in the promoted anatomy window. ENFORCE: `extension/tests/verify-recapture-fired.test.js`. PATTERN_SHAPE: `JSON\.parse\(fs\.readFileSync\(statePath` in `verify-recapture-fired.js`.
- `section-c-still-needed.js` — INVARIANT: Section C symptom scans MUST read the same watcher logs as `writeWatcherLivenessArtifact` (`tmux-runner.log` and `pipeline-runner.log`). BREAKS: Section C clears while AC-DR-05 still records `FEED TERMINATED`, so the bundle can report a false-clean watcher state. ENFORCE: extension/tests/section-c-gate.test.js. PATTERN_SHAPE: `tmux-runner\.log` without adjacent `pipeline-runner\.log` in watcher-symptom readers; INVARIANT: default session auto-discovery MUST rank sessions with watcher logs ahead of no-log dirs, then sort by watcher-log `mtime`. BREAKS: a newer idle or empty session masks the active log stream and flips Section C against the wrong session. ENFORCE: extension/tests/section-c-gate.test.js. PATTERN_SHAPE: `readdirSync\(DEFAULT_SESSIONS_DIR.*statSync\(dir\)\.mtimeMs|sort\(.*watcherLogMtime.*\)` without `hasWatcherLogs` precedence; INVARIANT: default session auto-discovery MUST resolve sessions from `getDataRoot()` so `PICKLE_DATA_ROOT` and future runtime-root changes stay aligned with runner-created sessions. BREAKS: custom data-root runs scan the wrong tree and default Section C to the wrong session or no session at all. ENFORCE: extension/tests/section-c-gate.test.js. PATTERN_SHAPE: `os\.homedir\(\).+pickle-rick.+sessions|DEFAULT_SESSIONS_DIR\s*=\s*path\.join\([^)]*sessions`; INVARIANT: Section C artifacts scoped to a session or runtime root MUST NOT overwrite tracked repo bundle files. BREAKS: session checks leak evidence across runs and dirty the worktree, blocking later pipeline launches. ENFORCE: extension/tests/section-c-gate.test.js. PATTERN_SHAPE: `path\.join\(REPO_ROOT,\s*'bundle',\s*'section-c-still-needed\.json'\)|const ARTIFACT_PATH = .*section-c-still-needed\.json`.
