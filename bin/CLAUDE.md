## Trap Doors

- `purge-update-cache.js` — INVARIANT: purge root MUST equal validated `getExtensionRoot()` resolution (`EXTENSION_DIR` only when install-root sentinel exists, else fallback to `~/.claude/pickle-rick`); `update-check.json` and `deploy-audit.log` are written by `extension/src/bin/check-update.ts` and `install.sh` to that path. BREAKS: purge no-ops in production when raw env paths or wrong defaults miss the live cache root; stale auto-updater cache poisons the next update check. ENFORCE: extension/tests/purge-update-cache.test.js. PATTERN_SHAPE: `process\.env\.EXTENSION_DIR\s*\|\||path\.join\(os\.homedir\(\), '\.claude', 'pickle-rick'\)`.
- `release-gate.sh` — INVARIANT: release tag semver MUST equal `extension/package.json` version before publish and after download verification. BREAKS: updater caches a phantom latest version and re-downloads the same tarball forever. ENFORCE: extension/tests/release-gate.test.js. PATTERN_SHAPE: `--pre-tag|--post-tag` path without tag-name-vs-package-version parity check; INVARIANT: release gating MUST resolve `extension/package.json` and git tag contents from the active worktree root, not the caller's nested cwd. BREAKS: `bash ../bin/release-gate.sh` from `extension/` or another repo subdir false-fails before validating a real release tag. ENFORCE: extension/tests/release-gate.test.js. PATTERN_SHAPE: `PKG_PATH=\"extension/package\.json\"|git show \"\\$tag:\\$PKG_PATH\"` without `git rev-parse --show-toplevel` root resolution.
- `verify-bundle.js` — INVARIANT: every `bundle/ac-dr-*.json` producer MUST emit the canonical bundle schema (`ac_id`, `checked_at`, `checker`, `checker_version`, `evidence`, `failure_reason`, `remediation_hint`). BREAKS: `verifyBundle()` rejects real session artifacts and the deploy-reversion bundle gate goes red before evaluating behavior. ENFORCE: extension/tests/verify-bundle.test.js, extension/tests/pipeline-runner.test.js. PATTERN_SHAPE: `write(StateFile|FileSync)\(.+bundle\/ac-dr-[^']+\.json`; INVARIANT: single-AC verification MUST normalize refined AC aliases to the canonical bundle artifact id before resolving `bundle/ac-dr-*.json`. BREAKS: `--ac AC-DR-15` false-fails against missing `ac-dr-15.json` while the full-bundle path expects `ac-dr-pre-flight.json`. ENFORCE: extension/tests/verify-bundle.test.js. PATTERN_SHAPE: `options\.ac\s*\?\s*\[options\.ac\]` without refined-id normalization.
- `verify-recapture-fired.js` — INVARIANT: AC-DR-02 MUST match `baseline_recapture_attempted` only inside the latest anatomy window and MUST NOT pin a fixed iteration. BREAKS: stale runs or later-iteration recaptures false-pass or false-fail the bundle. ENFORCE: extension/tests/verify-recapture-fired.test.js. PATTERN_SHAPE: `iteration\s*===\s*1|findMatchingEvent\(activity,\s*anatomyWindows\(state\.history\)\)` without latest-window-only filtering.
- `section-c-still-needed.js` — INVARIANT: Section C symptom scans MUST read the same watcher logs as `writeWatcherLivenessArtifact` (`tmux-runner.log` and `pipeline-runner.log`). BREAKS: Section C clears while AC-DR-05 still records `FEED TERMINATED`, so the bundle can report a false-clean watcher state. ENFORCE: extension/tests/section-c-gate.test.js. PATTERN_SHAPE: `tmux-runner\.log` without adjacent `pipeline-runner\.log` in watcher-symptom readers; INVARIANT: default session auto-discovery MUST sort by watcher-log `mtime`, not session-directory `mtime`. BREAKS: a newer idle session masks the active log stream and flips Section C against the wrong session. ENFORCE: extension/tests/section-c-gate.test.js. PATTERN_SHAPE: `readdirSync\(DEFAULT_SESSIONS_DIR.*statSync\(dir\)\.mtimeMs`.
