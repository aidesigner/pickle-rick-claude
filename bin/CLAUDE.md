## Trap Doors

- `purge-update-cache.js` — INVARIANT: purge root MUST equal `getExtensionRoot()` (`EXTENSION_DIR` env override, default `~/.claude/pickle-rick`); `update-check.json` and `deploy-audit.log` are written by `extension/src/bin/check-update.ts` and `install.sh` to that path. BREAKS: purge no-ops in production (commit 5fc4ecee shipped `.codex/pickle-rick` default which never matches the real cache location); stale auto-updater cache poisons the next update check. ENFORCE: extension/tests/purge-update-cache.test.js. PATTERN_SHAPE: `path.join(os.homedir(), '.codex', 'pickle-rick')`.
- `verify-bundle.js` — INVARIANT: every `bundle/ac-dr-*.json` producer MUST emit the canonical bundle schema (`ac_id`, `checked_at`, `checker`, `checker_version`, `evidence`, `failure_reason`, `remediation_hint`). BREAKS: `verifyBundle()` rejects real session artifacts and the deploy-reversion bundle gate goes red before evaluating behavior. ENFORCE: extension/tests/verify-bundle.test.js, extension/tests/pipeline-runner.test.js. PATTERN_SHAPE: `write(StateFile|FileSync)\(.+bundle\/ac-dr-[^']+\.json`.
- `section-c-still-needed.js` — INVARIANT: Section C symptom scans MUST read the same watcher logs as `writeWatcherLivenessArtifact` (`tmux-runner.log` and `pipeline-runner.log`). BREAKS: Section C clears while AC-DR-05 still records `FEED TERMINATED`, so the bundle can report a false-clean watcher state. ENFORCE: extension/tests/section-c-gate.test.js. PATTERN_SHAPE: `tmux-runner\.log` without adjacent `pipeline-runner\.log` in watcher-symptom readers; INVARIANT: default session auto-discovery MUST sort by watcher-log `mtime`, not session-directory `mtime`. BREAKS: a newer idle session masks the active log stream and flips Section C against the wrong session. ENFORCE: extension/tests/section-c-gate.test.js. PATTERN_SHAPE: `readdirSync\(DEFAULT_SESSIONS_DIR.*statSync\(dir\)\.mtimeMs`.
