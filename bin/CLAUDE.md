- `purge-update-cache.js` — INVARIANT: purge root MUST equal validated `getExtensionRoot()` resolution (`EXTENSION_DIR` only when install-root sentinel exists, else fallback to `~/.claude/pickle-rick`); `update-check.json` and `deploy-audit.log` are written by `extension/src/bin/check-update.ts` and `install.sh` to that path. BREAKS: purge no-ops in production when raw env paths or wrong defaults miss the live cache root; stale auto-updater cache poisons the next update check. ENFORCE: extension/tests/purge-update-cache.test.js. PATTERN_SHAPE: `process\.env\.EXTENSION_DIR\s*\|\||path\.join\(os\.homedir\(\), '\.claude', 'pickle-rick'\)`; INVARIANT: purge MUST delete both `pickle-update-*` download dirs and `pickle-extract-*` extraction dirs because `check-update.ts` uses both temp roots during release inspection/install. BREAKS: interrupted upgrades leave stale extracted payloads behind and operators get a false-clean purge. ENFORCE: extension/tests/purge-update-cache.test.js. PATTERN_SHAPE: `PKG_PATH=\"extension/package\.json\"|git show \"\\$tag:\\$PKG_PATH\"` without `git rev-parse --show-toplevel` root resolution; INVARIANT: post-tag verification MUST select the unique installable tarball by archive shape and require both `extension/package.json` and `install.sh`. BREAKS: sidecar assets or partial payloads false-green the release while updater install later fails on missing runtime files. ENFORCE: extension/tests/release-gate.test.js. PATTERN_SHAPE: `tar -tzf .*extension/package\.json` without a same-archive `install\.sh` sentinel check; INVARIANT: post-tag verification MUST require exactly one shared payload root containing both `extension/package.json` and `install.sh` before reading version metadata. PATTERN_SHAPE: `(^|/)extension/package\.json$` and `(^|/)install\.sh$` checked without shared-root intersection; INVARIANT: post-tag verification MUST download release assets via `gh release download --pattern '*.tar.gz'`, never the source archive flag.
