# Extension Subsystem

Compiled TS → JS lives in `extension/services/`, `extension/bin/`, `extension/hooks/`, `extension/types/`. Source in `extension/src/`. Tests in `extension/tests/` via `node --test`. Rebuild with `npx tsc` from `extension/`.

## Trap Doors

- `src/bin/council-publish.ts` — INVARIANT: every `execFileSync('gh', …)` passes `timeout`, threaded via `PublishOptions.ghTimeoutMs`. BREAKS: network hang stalls publish indefinitely. ENFORCE: hang-path tests using `__hang__` / `auth: 'hang'` / `hangOnCall` sentinels.
- `src/services/scope-resolver.ts` — INVARIANT: every `spawnSync('rg'|'grep', …)` in `findImporters` passes `timeout` (default `FIND_IMPORTERS_TIMEOUT_MS = 30_000`), threaded via `computeOneHop({ findImportersTimeoutMs })`; inline `diff:<ref>` scope bases must come from the flag itself, never from `scopeBase`/default-base fallback; per-phase `refreshScope()` must reapply the original `target` subtree clamp before persisting refreshed `allowed_paths`. BREAKS: wedged rg/grep stalls scope resolution silently; `diff:HEAD`/`diff:<release-branch>` silently reviews the wrong files and subsystems; later phase refreshes can widen scoped anatomy/szechuan review outside TARGET and misroute gate failures. ENFORCE: `scope-one-hop-hang-guard.test.js`, `scope-pipeline.test.js` diff:HEAD regression, and `scope-pipeline.test.js` target-clamp refresh regression.
- `src/bin/plumbus-frame-analyzer.ts` — INVARIANT: every `spawnSync('bun', …)` in `parseDotViaBun` passes `timeout` (default `BUN_TIMEOUT_MS = 30_000`). BREAKS: wedged bun stalls generative-audit pipeline silently. ENFORCE: `plumbus-frame-analyzer-hang-guard.test.js`.
- `src/services/pickle-utils.ts` — INVARIANT: macOS `osascript` shell-outs route through `displayMacNotification` (default `NOTIFICATION_TIMEOUT_MS = 5_000`); never call `spawnSync('osascript', …)` directly. BREAKS: wedged Notification Center blocks `process.exit`, leaks cancel markers, hangs tmux pane. ENFORCE: `notification-hang-guard.test.js` via `spawnSyncFn` seam.
- `src/bin/microverse-runner.ts` — INVARIANT: auto-commit rescue stages tracked + untracked files via the shared staging helper, honoring `docs/`/`prds/` exclusions; per-iteration convergence gates must capture `gate/baseline.json` from the clean pre-iteration tree before the first worker commit, never from the first post-change diff; stale baselines must be recaptured from the clean loop-start tree before reuse, honoring `baseline_max_age_iterations` / `baseline_max_age_seconds`; and they must baseline + replay `tests` alongside `typecheck`/`lint`. BREAKS: `git add -u`-only drops new test files and first-time `CLAUDE.md`, then misclassifies as stall; first anatomy regression gets silently accepted as the baseline and the gate false-greens the iteration that introduced it; resumed sessions can keep subtracting obsolete failures forever if stale `gate/baseline.json` is reused; pure test regressions slip through green when the per-iteration gate omits `tests`. ENFORCE: untracked-file path in `microverse.test.js` and first-regression bootstrap + stale-baseline refresh coverage in `anatomy-park-baseline-gate.test.js`.
- `src/bin/council-publish.ts` (directive/stack parity) — INVARIANT: every non-trunk branch in `council-stack.json` has a matching `directive.branches` entry; missing entry → `outcome: 'failed'`, no body file, no `.published` marker. BREAKS: silent "empty findings" comments hide dropped fan-out shards. ENFORCE: `stack branch absent from directive` test in `council-publish.test.js`.
- `src/bin/pipeline-runner.ts` — INVARIANT: scoped anatomy/szechuan phases pass `scope.json` into `init-microverse` so `microverse.json.allowed_paths` survives setup; when `refreshScope()` returns `null` because the phase was already entered on resume, setup must reuse the persisted `scope.json` instead of widening back to unscoped review. BREAKS: repo-wide gate failures get treated as in-scope and scoped resume runs silently expand from the target organ to the whole body. ENFORCE: `anatomy-park-scope.test.js` and `szechuan-scope.test.js`.
- `src/bin/finalize-gate.ts` — INVARIANT: synthetic gate failures (`<timeout>`, `<lock-timeout>`, `<workingdir-drift>`) stay in-scope even when `allowed_paths` is present; only real repo paths go through path-based scope filtering. BREAKS: scoped strict/final gates false-green when timeout/drift failures are misfiled as out-of-scope. ENFORCE: synthetic sentinel path in `anatomy-park-scoped-final-gate.test.js`.
- `src/bin/log-commit.ts` — INVARIANT: commit attribution must resolve the session for `process.cwd()` from `current_sessions.json` first, then fall back only to active sessions whose `state.json.working_dir` matches the same cwd; never pick the first active session globally. BREAKS: concurrent or cross-repo loops can stamp a commit onto the wrong session, and standup will confidently attribute that commit to the wrong task because the hook already set `event.session`. ENFORCE: multi-session cwd-mapping regression in `log-commit.test.js`.
- `src/bin/metrics.ts` — INVARIANT: user-visible/report date ranges are inclusive of the last represented day; if collection uses an internal upper bound, normalize it before writing `MetricsReport.until` or panel/no-data copy; CLI `--since` parsing must round-trip exact `YYYY-MM-DD` calendar components before scanning files, or impossible dates like `2026-02-30` normalize to March 2 and silently misreport the range; `--json` must always emit a `MetricsReport` object, even when there are zero rows. BREAKS: `--days 0` and default reports can claim tomorrow is in range even though no tomorrow rows were scanned; impossible `--since` dates silently shift the scan window to a different day; empty-data JSON callers crash on prose output instead of a parseable empty report. ENFORCE: CLI `--days 0`, impossible-calendar-date, and empty-report JSON regressions in `metrics.test.js`.
- `src/bin/standup.ts` — INVARIANT: `range.until` is an exclusive upper bound for activity-file selection and must never be printed directly; CLI `--since` parsing must round-trip exact `YYYY-MM-DD` calendar components before building the activity/git range, or impossible dates like `2026-02-30` normalize to March 2 and silently point the report at the wrong workday. BREAKS: standup headers and empty-range messages drift one day into the future while the reader still stops before that file; impossible `--since` dates silently shift the report window. ENFORCE: CLI `--days 0` and impossible-calendar-date regressions in `standup.test.js`.
- `src/services/pickle-utils.ts` — INVARIANT: data-root resolution must honor canonical `PICKLE_DATA_ROOT` first and legacy `PICKLE_DATA_DIR` second, with both feeding every `getDataRoot()` caller consistently; session discovery must treat `current_sessions.json` as a cache and fall back to `sessions/*/state.json` by `working_dir`, because setup can leave a live session behind when the map write fails or goes stale. BREAKS: setup, status, standup, hooks, and jar/session readers silently split across different storage roots when operators follow the documented override name; map-only readers lose live sessions, so cancel/status/retry/worker resume falsely report "no active session" while hooks and other state readers still see the run. ENFORCE: `getDataRoot: PICKLE_DATA_ROOT is the canonical override` in `pickle-utils.test.js`, the `PICKLE_DATA_ROOT` CLI path in `standup.test.js`, and missing-map fallback regressions in `cancel.test.js`, `get-session.test.js`, `retry-ticket.test.js`, `status.test.js`, and `worker-setup.test.js`.
- `src/services/pickle-utils.ts`; `src/bin/retry-ticket.ts` — INVARIANT: ticket resolution timestamps are state, not append-only audit junk: retrying a ticket back to `Todo` must clear `completed_at`/`skipped_at`, and later `Done`/`Skipped` transitions must replace existing values instead of duplicating frontmatter keys. BREAKS: retried tickets keep stale resolution metadata, subsequent completions append duplicate keys, and `parseTicketFrontmatter()` reads the stale first match. ENFORCE: retry timestamp reset regression in `retry-ticket.test.js` plus timestamp replacement coverage in `pickle-utils.test.js`.
- `src/services/activity-logger.ts` — INVARIANT: activity JSONL filenames are the storage partition key and must be derived from the event timestamp being written, not the wall clock at retry/flush time. BREAKS: caller-supplied historical timestamps and buffered writes that succeed after midnight land in the wrong file, and `standup.ts` drops or misdates them because it filters by filename before parsing lines. ENFORCE: `activity-logger.test.js` retroactive-timestamp regression that writes via `logActivity()` and reads back through `readActivityFiles()`.
- `src/services/convergence-gate.ts` — INVARIANT: scope matching must treat workspace package dirs and package-level test failures as in-scope when any `allowed_paths` entry lives under that package; exact file paths from `scope.json` are the normal case; repo-root workspace control files (`package.json`, `pnpm-workspace.yaml`, lockfiles) keep every discovered workspace package in scope because they can change package selection or commands globally; workspace discovery must expand nested workspace globs like `packages/*/*` all the way to the actual package directories before gating; project-type detection must prefer concrete lockfiles (`pnpm-lock.yaml`, `yarn.lock`, `package-lock.json`, `bun.lock`, `bun.lockb`) over generic `package.json`. BREAKS: strict/final gates silently skip in-scope package checks or mislabel failing package tests as out-of-scope; root workspace manifest scopes can collapse to zero packages and false-green a broken monorepo; nested workspace repos can false-green by discovering zero packages and running zero checks; Bun repos can false-green as npm, then the test safety filter skips `bun test` and no gate command runs. ENFORCE: `convergence-gate-workspaces.test.js` exact-file + root-control-file + nested-workspace fixtures, `anatomy-park-scoped-final-gate.test.js`, and `convergence-gate-resolution.test.js` bun+package.json case.
